# Start Debug Server

Start development server(s) with debug logging after killing existing processes. Auto-detects tech stack on first run.

## Arguments
- `$ARGUMENTS` - Optional: `backend`, `frontend`, `all`, or `detect` (default: `all`)

---

## First Run: Stack Detection

On first run (or when `detect` argument passed), analyze the project to detect:

### Detection Checks (run in parallel where possible)

| Check | Files | Detects |
|-------|-------|---------|
| **Package Manager** | `package.json`, `pnpm-lock.yaml`, `yarn.lock`, `bun.lockb` | npm/pnpm/yarn/bun |
| **Frontend Framework** | `package.json` deps, `vite.config.*`, `next.config.*`, `nuxt.config.*`, `angular.json` | Vite/Next.js/Nuxt/Angular/CRA |
| **Backend (Node)** | `package.json` scripts/deps for `express`, `fastify`, `hono`, `nest` | Express/Fastify/Hono/NestJS |
| **Backend (Python)** | `pyproject.toml`, `requirements.txt`, presence of `uvicorn`, `gunicorn`, `django`, `flask`, `fastapi` | FastAPI/Django/Flask |
| **Backend (Other)** | `go.mod`, `Cargo.toml`, `build.gradle`, `pom.xml` | Go/Rust/Java |
| **Containers** | `docker-compose.yml`, `docker-compose.yaml`, `Dockerfile` | Docker Compose/Docker |
| **Monorepo** | `pnpm-workspace.yaml`, `lerna.json`, `turbo.json`, `nx.json` | pnpm/Lerna/Turbo/Nx monorepo |
| **Port Config** | `vite.config.*`, `package.json` scripts, `.env*`, `docker-compose.yml` | Default ports |
| **Env Files** | `.env`, `.env.local`, `.env.development`, `backend/.env` | Required env vars |

After detection, output a summary:

```

```

## Workflow: Starting Servers

**⚠️ NON-BLOCKING EXECUTION RULES:**
- Use `timeout` command for all commands that could hang (lsof, network checks)
- Use `&` suffix or Bash tool's `run_in_background: true` for long-running processes
- Use `truncate` instead of `>` redirect for clearing large log files
- Add `|| true` to prevent failures from blocking the workflow
- Never use `tee` in foreground (blocks session) - redirect to files instead

### 1. Check for Existing Configuration

```bash
# Look for "## Debug Server Configuration" in CLAUDE.md
grep -q "## Debug Server Configuration" CLAUDE.md
```

- **If found**: Use cached configuration
- **If not found OR `detect` arg**: Run stack detection, generate config, save to CLAUDE.md

### 2. Kill Existing Servers (Non-Blocking)

Kill processes on detected ports with timeout to prevent blocking:
```bash
# Use timeout (5s max) to prevent lsof from hanging
timeout 5 lsof -ti:[port1] 2>/dev/null | xargs -r kill -9 2>/dev/null &
timeout 5 lsof -ti:[port2] 2>/dev/null | xargs -r kill -9 2>/dev/null &
wait  # Brief wait for kills to complete
sleep 1  # Allow ports to release
```

### 3. Ensure Log Directory (Non-Blocking)

```bash
mkdir -p [log_dir]
# Clear logs non-blocking: truncate is faster than redirect for large files
truncate -s 0 [log_dir]/*.log 2>/dev/null || rm -f [log_dir]/*.log &
```

### 4. Verify Environment

Check that required env files and variables exist. Warn if missing.

### 5. Start Server(s)

**IMPORTANT: All startup commands MUST use `run_in_background: true` with the Bash tool to prevent session blocking.**

Based on detected stack type:

#### Frontend-Only (Vite/React/Vue)
```bash
# Run with Bash tool's run_in_background: true
LOG_LEVEL=debug npm run dev >> [log_dir]/frontend.log 2>&1
```

#### Fullstack (Separate Backend + Frontend)
```bash
# Backend (run_in_background: true)
cd [backend_dir] && [activate_venv_if_python] && \
  LOG_LEVEL=DEBUG [backend_cmd] >> [log_dir]/backend.log 2>&1

# Frontend (run_in_background: true, separate call)
cd [frontend_dir] && LOG_LEVEL=debug [frontend_cmd] >> [log_dir]/frontend.log 2>&1
```

#### Docker-Based
```bash
# run_in_background: true
docker compose -f [compose_file] up --build >> [log_dir]/docker.log 2>&1
```

#### Monorepo
```bash
# run_in_background: true
[package_mgr] run dev >> [log_dir]/monorepo.log 2>&1
```

### 6. Output Status Banner

```

===========================
```

## Stack-Specific Templates

### Vite + React/Vue (Frontend-Only)
```yaml
type: frontend-only
frontend:
  framework: Vite
  command: npm run dev
  port: 5173
  dir: .
log_dir: ./logs
health_check: curl http://localhost:5173
```

### Next.js
```yaml
type: frontend-only  # or fullstack with API routes
frontend:
  framework: Next.js
  command: npm run dev
  port: 3000
  dir: .
log_dir: ./logs
health_check: curl http://localhost:3000/api/health
```

### FastAPI + Vite (Fullstack)
```yaml
type: fullstack
backend:
  framework: FastAPI
  command: uvicorn main:app --host 0.0.0.0 --port 8000 --reload
  port: 8000
  dir: backend
  venv: backend/venv
  env_file: backend/.env
frontend:
  framework: Vite
  command: npm run dev
  port: 5173
  dir: frontend
log_dir: backend/logs
health_check: curl http://localhost:8000/health
```

### Express + React (Fullstack)
```yaml
type: fullstack
backend:
  framework: Express
  command: npm run dev
  port: 3001
  dir: server
frontend:
  framework: Vite
  command: npm run dev
  port: 5173
  dir: client
log_dir: ./logs
```

### Docker Compose
```yaml
type: docker-based
compose_file: docker-compose.yml
services:
  - name: backend
    port: 8000
  - name: frontend
    port: 3000
  - name: db
    port: 5432
log_dir: ./logs
health_check: docker compose ps
```

## Usage Examples

```bash
/start-debug-server              # Start all servers using cached/detected config
/start-debug-server detect       # Force re-detection and update CLAUDE.md
/start-debug-server frontend     # Start frontend only
/start-debug-server backend      # Start backend only
/start-debug-server all          # Explicitly start all (same as no args)
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Port conflict | `timeout 5 lsof -ti:[port] \| xargs -r kill -9` |
| lsof hanging | Use `timeout 5 lsof ...` to prevent blocking |
| Log clearing slow | Use `truncate -s 0 file.log` instead of `> file.log` |
| Session blocked | Use Bash tool with `run_in_background: true` for dev servers |
| Wrong stack detected | Edit `## Debug Server Configuration` in CLAUDE.md manually |
| Config outdated | Run `/start-debug-server detect` to re-analyze |
| Venv not found | `cd [backend_dir] && python -m venv venv && pip install -e ".[dev]"` |
| Node deps missing | `[pkg_mgr] install` in relevant directory |
| Docker not running | `docker compose up -d` or start Docker Desktop |

---

## Notes

- **All CLI commands must be non-blocking** - use `timeout`, `&`, or `run_in_background: true`
- Configuration cached in CLAUDE.md survives sessions
- Re-run with `detect` argument to refresh configuration
- Logs are cleared on each startup using `truncate` (faster than redirect for large files)
- For monorepos, detection walks workspace packages
- Python venvs are auto-activated when detected
- Frontend proxying to backend is framework-specific (check vite.config or next.config)
- Dev servers should always run with `run_in_background: true` to avoid session blocking
